# CodexLight 使用与实现说明

[English](USAGE_AND_IMPLEMENTATION.md) | 简体中文 | [返回文档索引](README.md) | [项目主页](../README.md)

本文合并项目操作手册和技术实现说明，覆盖固件烧录、USB Wi-Fi 配网、有线/无线运行、排障和维护。

## 硬件连接

| LED | 数据引脚 | 状态 |
| --- | --- | --- |
| 黄色 WS2812B | GPIO5 | 未连接、等待或诊断 |
| 绿色 WS2812B | GPIO6 | 连接动画或任务完成 |
| 红色 WS2812B | GPIO7 | Codex 正在工作或 Wi-Fi 重连诊断 |

三颗 WS2812B 分别使用独立 DIN。VCC 接稳定 5 V，GND 与 ESP32-C3 共地。

## 编译与烧录

```powershell
cd C:\path\to\CodexLight\Firmware
pio run
pio run -t upload --upload-port COM4
pio device monitor --port COM4 --baud 115200
```

启动 Bridge 前请关闭 PlatformIO Monitor，因为它们会占用同一个串口。

## USB Wi-Fi 配网

当前固件不会开启 SoftAP 配网页面。配网通过 USB 串口完成。

托盘流程：

1. USB 连接设备。
2. 启动 `Bridge\start_codex_light_tray.vbs`，该启动器默认进入 `WIRELESS` 模式。
3. 右键托盘图标，选择 `Configure WiFi`。
4. 输入 SSID/密码并保存。

命令行流程：

```powershell
python Bridge\codex_light_monitor.py --serial auto --wifi-ssid "YourWifi" --wifi-password "YourPassword"
```

固件命令：

```text
WIFI_SET <ssid><TAB><password>
```

只有当 `WiFi.status() == WL_CONNECTED` 时，固件才保存 Wi-Fi。失败尝试不会写入 Flash。

## 通信模式

| 模式 | 行为 |
| --- | --- |
| `WIRED` | 只接收 USB 串口心跳 |
| `WIRELESS` | 只接收 UDP 心跳 |
| `AUTO` | 同时接收 USB 和 UDP；USB 心跳新鲜时优先 |

模式命令：

```text
MODE WIRED
MODE WIRELESS
MODE AUTO
STATUS
```

模式保存在 Preferences namespace `codexlight` 的 `transport` 键中。

## Windows 托盘

使用隐藏启动器，默认进入 `WIRELESS` 模式：

```text
Bridge\start_codex_light_tray.vbs
```

托盘提供：

- `Configure WiFi`：USB 配网。
- `Connection mode`：切换 AUTO、WIRED、WIRELESS。
- `Open log folder`：打开 `Bridge/logs`。
- `Restart monitor`：重启 `codex_light_monitor.py`。
- `Exit`：退出托盘。

无线托盘路径在 USB 存在时只短暂打开串口，发送 reset pulse 和 `MODE WIRELESS` 后释放端口。没有 USB 串口时，它会记录 `SERIAL setup skipped; using saved firmware mode.`，然后继续通过 UDP 发送状态。

## 总体架构

```text
Codex Desktop logs
        |
        v
Bridge/codex_light_monitor.py
        |
        +-- USB CDC serial: GREEN / RED / YELLOW / MODE / WIFI_SET
        +-- LAN UDP: CODEXLIGHT/1 <STATE> and CODEXLIGHT/1 PING
                         |
                         v
                  ESP32-C3 firmware
                         |
                         +-- Preferences: wifi ssid/password, transport mode
                         +-- non-blocking saved Wi-Fi reconnect
                         +-- GPIO5 yellow WS2812B
                         +-- GPIO6 green WS2812B
                         +-- GPIO7 red WS2812B
```

## Bridge 实现

Bridge 读取：

- `~/.codex/sessions/**/*.jsonl`
- `~/.codex/logs_2.sqlite`

它只向设备发送颜色状态。

| Codex 事件 | 输出 |
| --- | --- |
| `task_started`、消息、推理、工具调用、工具输出 | `RED` |
| 需要审批、权限或用户输入 | `YELLOW` |
| `task_complete` 或 `turn_aborted` | `GREEN` |

托盘配网会暂停监控，运行一次性 `--wifi-config` 子进程，把日志写到 `Bridge/logs/wifi_setup.*.log`，然后恢复监控。

## 固件实现

主要文件：

- `Firmware/src/main.cpp`：串口命令解析、UDP 心跳、传输选择、LED 状态机、诊断灯码。
- `Firmware/src/config_portal.cpp`：STA Wi-Fi 连接、USB 配网封装、已保存 Wi-Fi 的非阻塞重连。
- `Firmware/src/storage.cpp`：Wi-Fi 凭据 Preferences 存储。
- `Firmware/src/led.cpp`：三路独立 NeoPixel 输出。
- `Firmware/include/config.h`：GPIO、亮度、超时、UDP 端口、默认模式和 ESP32-C3 Wi-Fi 发射功率。

Wi-Fi 说明：

- 固件只使用 USB 串口配网。
- 只有 STA 到达 `WL_CONNECTED` 后才保存凭据。
- 已保存 Wi-Fi 在启动时非阻塞连接，即使路由器不可达，LED 诊断和主循环也会继续运行。
- 已保存 Wi-Fi 失败时保留凭据，并每 10 秒重试。
- ESP32-C3 发射功率限制为 `WIFI_MAX_TX_POWER_QDBM = 34`，即 8.5 dBm，以提升小型开发板的无线稳定性。
- 默认关闭调试串口输出，便于独立供电无线运行。

启动流程：

```text
setup
  -> 初始化 LED 并显示启动黄灯
  -> 读取已保存 Wi-Fi 凭据
  -> 如果存在，非阻塞启动 STA 连接
  -> 读取已保存通信模式
loop
  -> 需要时继续尝试 Wi-Fi 重连
  -> 处理可用 USB 串口命令
  -> Wi-Fi 已连接时维护 UDP
  -> 选择活动传输
  -> 渲染 LED 状态或无串口诊断灯码
```

```text
CODEXLIGHT/1 HELLO mac=<MAC> mode=<MODE>
```

## 常见问题

- Wi-Fi 配网失败：查看 `Bridge/logs/wifi_setup.out.log` 和 `.err.log`，关闭 PlatformIO Monitor，确认使用 2.4 GHz Wi-Fi，并确认固件使用 `WIFI_MAX_TX_POWER_QDBM = 34`。
- 无线不更新：确认同一局域网、防火墙允许 UDP 4210，并重启托盘重新发现设备。
- 黄灯慢闪：Wi-Fi 已连接，固件正在等待电脑心跳；打开托盘。
- 红色双闪：有保存的 Wi-Fi，但连接失败或正在重连。
- 颜色错误：确认 `NEO_GRB`、DIN 方向、GPIO 连通性和 5 V 供电。

## 安全

- Wi-Fi 凭据保存在 ESP32 NVS，固件不会通过串口打印密码。
- UDP 未加密也未鉴权，只应在可信局域网使用。
