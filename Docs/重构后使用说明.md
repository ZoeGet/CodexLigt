# CodexLight 重构后使用说明

[返回文档索引](README.md) | [项目主页](../README.md)

本文是重构后版本的中文操作手册。当前实现支持 USB 串口和同一局域网内的 UDP，Wi-Fi 通过手机连接 ESP32 AP 后在网页中配置。

## 1. 硬件连接

| LED | 数据引脚 |
| --- | --- |
| 黄灯 WS2812B | GPIO5 |
| 绿灯 WS2812B | GPIO6 |
| 红灯 WS2812B | GPIO7 |

三颗灯分别使用独立 DIN。VCC 接稳定 5 V，GND 与 ESP32-C3 共地。

## 2. 编译与烧录

```powershell
cd C:\path\to\CodexLight\Firmware
pio run -j 1
pio run -t upload --upload-port COM4
```

替换实际 COM 端口。烧录完成后查看串口：

```powershell
pio device monitor --port COM4
```

正常输出：

```text
CODEXLIGHT READY
STATUS mode=WIRED active=NONE wifi=DISCONNECTED
```

## 3. 仅使用有线串口

1. 使用 `Ctrl+C` 关闭 PlatformIO 串口监视器。
2. 回到仓库根目录。
3. 启动 Bridge：

```powershell
python -m pip install pyserial
python Bridge\codex_light_monitor.py --serial COM4 --baud 115200
```

Bridge 连接后会自动发送 `MODE WIRED`。ESP32 首次收到心跳时绿灯闪烁 2 秒，随后显示实际状态。

## 4. 手机 AP 配网

没有可用 Wi-Fi 凭据时，ESP32 开启：

```text
SSID: CodexLight-XXXX
Password: 123456789
URL: http://192.168.4.1
```

手机连接热点后，在配置页面选择电脑也能连接的 2.4 GHz Wi-Fi，输入密码并保存。ESP32-C3 不支持 5 GHz Wi-Fi。

需要重新配网时，在串口中发送：

```text
WIFI_CONFIG
CLEAR_WIFI
```

`WIFI_CONFIG` 打开配网页；`CLEAR_WIFI` 删除已有凭据后打开配网页。

## 5. 使用无线 UDP

先确保电脑和 ESP32 位于同一局域网，然后通过串口发送：

```text
MODE WIRELESS
```

关闭串口监视器，启动无线 Bridge：

```powershell
python Bridge\codex_light_monitor.py --udp --udp-port 4210
```

Bridge 会通过广播发现 ESP32，并将最近 IP 保存到 `Bridge/config.local.json`。该文件不会提交到 Git。

## 6. 同时启用两种连接

```powershell
python Bridge\codex_light_monitor.py --serial COM4 --baud 115200 --udp --udp-port 4210
```

Bridge 检测到串口和 UDP 同时启用时会自动发送 `MODE AUTO`。`AUTO` 模式下，有效的有线心跳优先；有线心跳失效后可以使用无线心跳。

## 7. 通信模式和 NVS

```text
MODE WIRED
MODE WIRELESS
MODE AUTO
STATUS
```

模式会保存到 NVS。重新烧录固件通常不会删除 NVS。如果需要完全擦除：

```powershell
cd Firmware
pio run -t erase --upload-port COM4
pio run -t upload --upload-port COM4
```

## 8. LED 行为

- 未收到有效心跳：GPIO5 黄灯按 1 秒周期闪烁。
- 建立电脑端连接：GPIO6 绿灯闪烁 2 秒。
- 工作或执行工具：GPIO7 红灯常亮。
- 等待批准或用户输入：GPIO5 黄灯常亮。
- `task_complete` 或 `turn_aborted`：GPIO6 绿灯常亮。
- 超过 6 秒未收到当前传输的心跳：回到黄灯闪烁。

## 9. 常见问题

### 黄灯一直闪

- Bridge 未启动或串口被其他程序占用。
- 固件模式是 `WIRELESS`，但当前只运行串口 Bridge。
- Bridge 使用了错误 COM 端口。
- 无线模式下电脑和 ESP32 不在同一局域网。

### 手机连接热点失败

- 确认 AP 密码至少 8 位且与 `config.h` 一致。
- 暂时关闭手机的自动网络切换。
- 手动访问 `http://192.168.4.1`。
- 发送 `CLEAR_WIFI` 后重试。

### 绿色显示成蓝色

当前板与参考项目一致，使用 `NEO_GRB` 色序。其他灯珠批次可能需要修改 `Firmware/src/led.cpp` 中的 NeoPixel 色序。

### 工作中反复变绿

重启 Bridge，确保使用最新脚本。当前版本只在 `task_complete` 或 `turn_aborted` 后切换绿色。

## 10. 配置入口

固件参数统一位于：

```text
Firmware/include/config.h
```

常用可修改项包括 LED 引脚、亮度、UDP 端口、心跳超时、AP 名称和密码、默认通信模式。

完整说明见仓库根目录 [README.md](../README.md)。
