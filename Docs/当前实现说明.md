# CodexLight 当前实现说明

[返回文档索引](README.md) | [项目主页](../README.md)

本文记录当前代码的架构和关键行为，用于后续维护。用户操作以根目录 `README.md` 为准。

## 总体架构

```text
Codex Desktop logs
        │
        ▼
Bridge/codex_light_monitor.py
        │
        ├─ USB CDC serial: GREEN / RED / YELLOW
        └─ LAN UDP: CODEXLIGHT/1 <STATE>
                         │
                         ▼
                  ESP32-C3 firmware
                         │
                         ├─ GPIO5 yellow WS2812B
                         ├─ GPIO6 green WS2812B
                         └─ GPIO7 red WS2812B
```

## Bridge

入口文件：`Bridge/codex_light_monitor.py`

主要数据源：

- `~/.codex/sessions/**/*.jsonl`
- `~/.codex/logs_2.sqlite`

当前状态规则：

| 事件 | 状态 |
| --- | --- |
| `task_started` | `RED` |
| `agent_message`、`agent_reasoning`、`token_count` | `RED` |
| `function_call`、`function_call_output` | `RED` |
| 调用参数包含批准、权限或用户输入标记 | `YELLOW` |
| `task_complete` | `GREEN` |
| `turn_aborted` | `GREEN` |

活动任务期间不会因 `item/completed` 或静默超时变绿。`quiet_timeout` 只在没有活动 turn 时作为回退。

串口连接后 Bridge 等待 2 秒：仅串口时发送 `MODE WIRED`，串口和 UDP 同时启用时发送 `MODE AUTO`。随后发送当前状态，并每 2 秒重发一次作为固件连接心跳。

UDP 初始广播，收到匹配设备的 `HELLO` 后记录 IP。发现信息写入 `Bridge/config.local.json`，该文件被 Git 忽略。

## 固件

入口文件：`Firmware/src/main.cpp`

主要模块：

- `config_portal.cpp`：WiFiManager 非阻塞 AP 配网
- `led.cpp`：三路独立 Adafruit NeoPixel 驱动
- `config.h`：用户配置

固件主循环持续执行：

1. 解析 USB 串口命令。
2. 调用 WiFiManager `process()`。
3. 管理 Wi-Fi UDP 监听和 `HELLO` 广播。
4. 根据模式和心跳选择活动传输。
5. 更新连接动画或真实 LED 状态。

流程中没有等待 Wi-Fi 的无限阻塞循环，也没有 `while (!Serial)`，因此无网络或未连接电脑时仍能独立运行。

## 通信模式

- `WIRED`：只使用 `lastWiredPacketMs`。
- `WIRELESS`：只使用 `lastWirelessPacketMs`。
- `AUTO`：两者都有效时优先有线。

模式通过 Preferences namespace `codexlight` 的 `transport` 键保存在 NVS。默认模式是 `WIRED`。

## 心跳与 LED

- Bridge 心跳间隔：2 秒。
- 固件连接超时：6 秒。
- 断开闪烁半周期：500 ms，即完整周期 1 秒。
- 建立连接动画：2 秒。
- 连接动画闪烁半周期：250 ms。

LED 驱动与参考项目一致，使用 `NEO_GRB + NEO_KHZ800`。每次状态切换都会分别向三路灯发送数据，关闭非活动灯。

## Wi-Fi 配网

WiFiManager 在无可用凭据时开启 `CodexLight-XXXX` 热点，地址为 `192.168.4.1`。当前密码为 `123456789`。

`WIFI_CONFIG` 调用 `startConfigPortal()`；`CLEAR_WIFI` 调用 `resetSettings()`、清除连接并重新开启 Portal。

ESP32-C3 使用 STA 连接家庭或办公 Wi-Fi。手机 AP 只用于写入目标 Wi-Fi 凭据，不是电脑与设备长期通信的网络。

## 当前协议

串口：

```text
GREEN
RED
YELLOW
PING
STATUS
MODE WIRED
MODE WIRELESS
MODE AUTO
WIFI_CONFIG
CLEAR_WIFI
```

UDP：

```text
CODEXLIGHT/1 GREEN
CODEXLIGHT/1 RED
CODEXLIGHT/1 YELLOW
CODEXLIGHT/1 PING
```

发现：

```text
CODEXLIGHT/1 HELLO mac=<MAC> mode=<MODE>
```

## 已移除的旧实现

以下内容不再属于当前实现：

- `wifi_secrets.h` 硬编码 Wi-Fi 凭据
- 自定义阻塞配网页
- UDP token 配对和 `PAIR_SET`
- FastLED 多通道全局刷新
- 使用普通 `item/completed` 提前判断整轮任务完成
